# Program Output

## Introduction

This document demonstrates the expected output of the `MySqlDemo.java` application. It explains the console messages, database changes, SQL execution results, and verification steps that occur after successfully running the program.

Understanding the output is essential because it confirms that the Java application, JDBC API, and MySQL database are communicating correctly.

---

# Purpose

The objectives of this document are to:

- Explain every output generated by the application.
- Verify successful database connectivity.
- Verify SQL statement execution.
- Demonstrate expected database changes.
- Help troubleshoot unexpected outputs.

# Console Output 1: Database Connection

## Example

```text
Connected Successfully
```

## Explanation

This message indicates that:

- The JDBC Driver was loaded successfully.
- The database URL is correct.
- The username and password are valid.
- The MySQL Server is running.
- A `Connection` object has been created.

# Console Output 4: Rows Affected

Example:

```text
Rows Affected : 1
```

## Explanation

The integer returned by `executeUpdate()` indicates how many rows were modified.

| Value | Meaning |
|--------|----------|
| 0 | No rows affected or DDL statement executed |
| 1 | One row inserted, updated, or deleted |
| N | N rows modified |

# Possible Error Outputs

## Invalid Username or Password

```text
Access denied for user 'root'
```

### Cause

Incorrect database credentials.

### Solution

Verify the username and password.

---

## Unknown Database

```text
Unknown database 'sampledb'
```

### Cause

Database has not been created.

### Solution

Create the required database before running the application.

---

## Missing JDBC Driver

```text
No suitable driver found
```

### Cause

MySQL Connector/J is not configured.

### Solution

Add the JDBC Driver to the project.

---

## MySQL Server Not Running

```text
Communications link failure
```

### Cause

MySQL Server is stopped.

### Solution

Start the MySQL Server service.
